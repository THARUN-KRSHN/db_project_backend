# Smart Inventory & Billing - Backend API

## Overview
The backend of the Smart Inventory & Billing system is built with **FastAPI** to provide a high-performance, robust RESTful API. It handles multi-tenant shop management, intricate inventory tracking, automated billing, and role-based access control leveraging **Supabase Authentication**.

## Key Features
- **Multi-Tenant Architecture**: Supports multiple independent shops on a single platform ensuring complete data isolation.
- **Role-Based Access Control (RBAC)**: Distinct permissions for Shop Administrators and Staff members using Supabase JWT verification.
- **Inventory Management**: Full CRUD operations for products, bulk importing via CSV, and low-stock threshold alerting.
- **Sales & Billing**: Real-time invoice generation and daily revenue tracking.
- **Public Storefront API**: Exposes selective read-only inventory data for customer-facing public storefronts.
- **Dashboard Analytics**: Calculates real-time total revenue, item sales, and stock shortages.

## Tech Stack
- **Framework**: FastAPI
- **Language**: Python 3.10+
- **Database**: PostgreSQL
- **ORM**: SQLAlchemy
- **Authentication**: Supabase (GoTrue via `supabase-py`)
- **Server**: Uvicorn

## Environment Setup
Create a `.env` file in the `backend` directory:
```env
# Supabase Configuration
SUPABASE_URL=your_supabase_url
SUPABASE_ANON_KEY=your_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key

# PostgreSQL Configuration
DATABASE_URL=postgresql://user:password@host:port/dbname
```

## Running Locally
1. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
2. Start the server:
   ```bash
   uvicorn main:app --reload
   ```
3. Access interactive API documentation automatically generated by Swagger at: `http://localhost:8000/docs`

## Core Routes
- `/auth/`: Registration, Login, Staff Management, Staff Bulk Operations
- `/shops/`: Shop settings, profile image uploads
- `/inventory/`: Product management, bulk-import
- `/sales/`: Checkout processing, daily billing statistics
- `/dashboard/`: Analytical queries for admin panels
- `/public/`: Open endpoints for the public-facing shop catalogs
